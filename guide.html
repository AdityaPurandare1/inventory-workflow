The system takes two data sources — Slack messages from your counting teams and Craftable variance reports — merges them, fuzzy-matches products, scores severity, and produces a recount decision list the counting team can act on immediately.

Everything runs in the browser. Your API keys stay in your browser's local storage — nothing is sent to any server except the Craftable API and GitHub Models API directly.

First-Time Setup
~10 minutes

Deploy to GitHub Pages

Create a new repo on GitHub and upload the index.html file.

# From your terminal or just drag-drop in GitHub git init inventory-workflow cd inventory-workflow # Add index.html to this folder git add . git commit -m "Deploy" git branch -M main git remote add origin https://github.com/YOUR_USER/inventory-workflow.git git push -u origin main

Then in your GitHub repo:

Settings → Pages → Source: main / root → Save

Your app will be live at https://YOUR_USER.github.io/inventory-workflow/

Add Your GitHub Token

This powers the AI parsing of Slack messages via GPT-4o-mini. Uses your existing GitHub account.

1. Go to github.com/settings/tokens
2. Click Generate new token (classic)
3. No special scopes needed — just create it and copy the token (starts with ghp_...)

In the app:

Settings tab → GitHub Token → Paste → Save

Add Your Craftable API Key

This connects to Craftable for automated variance pulls.

In the app:

Settings tab → Craftable API section

Fill in:

API Base URL: https://api.craftable.com/v1 API Key: PytYKiR0BdqOnhCIy_6sG0jhXO3FFBAPt_o82Bx (your actual key) Auth Method: Authorization: Bearer <key> (try this first)

Click Save API Config.

Not sure about the auth method? Try Bearer first. If the API Explorer shows 401/403 errors, switch to X-API-Key, then query param.

Discover API Endpoints

This is the critical step — it scans the Craftable API and auto-configures which endpoints to use.

API Explorer tab

Click the 4 buttons in order:

1. Discover Endpoints — scans swagger spec, finds all API paths 2. Test Auth — verifies your key works 3. List Locations — pulls your venues from Craftable, syncs them 4. Test Variance — pulls a sample variance report, shows data structure

The log window shows everything happening in real-time. Green = success, red = error.

If you see CORS errors: This means Craftable's API doesn't allow browser-to-API calls from your GitHub Pages domain. Two solutions:

Option A: Use "Import CSV" on the Craftable tab — export variance from Craftable's web UI and upload it.
Option B: Open the Swagger UI directly at api.craftable.com/v1/swagger-ui/index.html, look at the endpoint names, and type them manually into Settings → Discovered Endpoints.
What the endpoints look like: After discovery, go to Settings → Discovered Endpoints. You should see something like:

Locations: /locations or /establishments
Variance: /variance or /reports/variance
Products: /products or /items

If the auto-discovery missed one, type it in manually from the Swagger UI.

Enable Automation

Set the system to auto-pull from Craftable on a schedule.

Settings tab → Automation section

Pull Interval: Every 30 minutes (recommended for count days) Min Variance $: 50 (skip items below this)

Click Save & Start. The green pulsing dot in the header confirms it's running.

The auto-pull runs while the browser tab is open. Keep it open on a counting day and it will refresh data every 30 minutes automatically. The Recount Report tab updates live.

Daily Operations
Count Day Workflow

Parse Slack Messages

As counting teams post messages in their Slack channels, paste them into the app.

Parse tab → Select venue → Paste messages → Parse & Add Issues

The AI extracts:

Product name Patrón Silver 750ml Issue type Missing, Damaged, No Sticker, Not Scanning, etc. Quantity 3 bottles Location Back Bar, Walk-In, Service Well, etc. Notes Brief summary of the issue

You can paste messages from multiple staff members at once — the AI handles multi-message parsing.

Adding New Venues On-the-Fly

If you need a venue that's not in the list:

Parse tab → Click "+ New" → Type name → Add

Or add in bulk:

Settings tab → Venues → Type name → + Add

The pre-loaded venues match your current H.Wood locations: Bird Streets Club, Delilah - LA, Deliliah - Miami, Keys, Poppy, The Nice Guy.

Pull Craftable Variance

If automation is running, this happens automatically. To pull manually:

Craftable tab → Select venue (or All) → Set min variance → Pull Now

If using CSV import instead:

Craftable tab → Import CSV → Select your Craftable export file

The system auto-detects columns from the CSV (product name, variance $, variance %, category, store).

Review Recount Report

This is the main output — where the counting team gets their action list.

Recount tab

Dashboard: Shows total issues, recount count, critical count, total variance.

Filters: Narrow by venue, severity, or "Recount Needed" only.

Sort: Click any column header to sort. Click again to reverse.

What the columns mean:

Venue Which H.Wood location Product Item name (+ Craftable match status underneath) Issue Missing, Damaged, No Sticker, Unknown variance, etc. Location Where in the venue (Back Bar, Walk-In, etc.) Qty Bottles short Var $ Dollar variance from Craftable (red = negative) Wks How many weeks this item has been flagged Severity Auto-scored (see reference below) Recount? RECOUNT = team needs to verify this item

Export for the Counting Team

Two export formats:

Recount CSV — Streamlined. Only items that need recounting. Includes a summary footer with totals per venue. Give this to the counting team.

Recount tab → ↓ Recount CSV

Full CSV — Everything. All items, all columns including SKU, category, repeat status, Craftable match confidence, timestamps. For your own records or deeper analysis.

Recount tab → ↓ Full CSV

The Recount CSV opens cleanly in Excel. It includes a BOM header for proper character encoding, quoted fields to prevent comma issues, and a summary section at the bottom with per-venue breakdowns.

Severity & Recount Logic

How Severity is Scored

Items are auto-scored based on how long they've been flagged, whether they're still active, and the dollar variance from Craftable.

SeverityCriteriaRecount?
CRITICAL Active 10+ weeks, or active 5+ weeks with $200+ variance Always
HIGH Active 5+ weeks, or inactive 5+ weeks with $500+ variance Always
MEDIUM Active 3+ weeks, or active with variance above threshold If variance ≥ $100 or weeks ≥ 3
WATCH Active, flagged 2 weeks No
LOW New items, or inactive with low variance No
Thresholds are configurable. Go to Settings → Recount Thresholds to change the minimum variance $ and minimum weeks required for a MEDIUM item to trigger a recount.

How Cross-Referencing Works

When Craftable data is pulled, the system fuzzy-matches each parsed issue against Craftable products. This handles the real-world typos in your data (like "Minuty Presige" matching to "Minuty Prestige", or "Mersault" matching "Meursault").

Each item gets a Craftable status:

YES — In Craftable 85%+ match confidence REVIEW — Low Confidence 60-85% match (check manually) NOT IN CRAFTABLE No match found

Troubleshooting

CORS Errors on Craftable API

If the API Explorer shows "CORS or connectivity issue", the Craftable API server doesn't allow browser requests from your GitHub Pages domain.

Fix: Use the CSV import workflow instead. Export variance from Craftable's web dashboard, then upload it in the Craftable tab → Import CSV.

The cross-referencing, severity scoring, and recount flagging all work identically with imported CSV data.

401 Unauthorized / 403 Forbidden

Your API key isn't being sent correctly.

Try switching the auth method in Settings → Auth Method:

Try in this order: 1. Authorization: Bearer <key> (most common) 2. X-API-Key: <key> (some APIs use this) 3. ?api_key=<key> (query parameter fallback)

AI Says "No Issues Found"

The AI couldn't extract inventory issues from your text. Tips:

• Include product names and quantities in the messages
• Mention specific issues: "missing", "damaged", "can't scan", "wrong label"
• More context is better: "3 bottles of Patrón Silver missing from back bar" works much better than "Patrón short"

Data Disappeared After Closing Browser

All data is stored in your browser's localStorage. It persists across sessions unless you clear your browser data, use incognito/private mode, or switch browsers/devices.

Best practice: Export your CSV at the end of each count day. The CSV is your permanent record.

Auto-Pull Stopped Working

The auto-pull only runs while the browser tab is open and active. If you close the tab or put your laptop to sleep, it pauses.

Reopen the app and the green dot should re-appear. If not, go to Settings → Automation → Save & Start to restart it.
